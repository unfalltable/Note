## 以太坊问题

- 以太坊中状态树、交易树、收据树的区别

  |      | 状态树       | 交易树           | 收据树           |
  | ---- | ------------ | ---------------- | ---------------- |
  | 内容 | 全部用户信息 | 当前区块交易信息 | 当前区块收据信息 |
  |      |              |                  |                  |
  |      |              |                  |                  |

- 以太坊中为什么要使用状态树
  - 因为ETH有账户的性质，需要保存账户的信息
  - 使用hash表的话
    - 每次每次构建merkle tree时都需要维护账户信息，代价太大
  - 使用比特币相同的merkle的话
    - 账户更新复杂，查找效率低
    - 因为需要维护账户信息，所以是否排序也是个问题
      - 排序：插入代价较大
      - 不排序：每个全节点的树结构的不唯一
- 以太坊中可以向不存在的地址转账吗
  - 可以的
